# 算力冗余：推理写作逻辑 (Reasoning Logic)

本文件定义了 Agent 在生成深度内容时的“思考路径”，旨在消除 AI 味。

## 1. Step-by-Step 分解 (Decomposition)
Agent 在动笔前，必须先在 `<think>` 中完成以下分解：
- **核心命题**：这篇文章要解决什么“不为人知”的技术矛盾？
- **利益相关者**：谁是赢家？谁是输家？谁在虚张声势？
- **数据支点**：哪一个技术参数（如：LPDDR5x 带宽、H100 功耗）是决定性的？

## 2. “思维预演”框架 (Chain-of-Thought)
在生成大纲后，Antagonist 角色必须执行“二次证伪”：
- **反向假设**：如果这个技术趋势失败了，最可能的原因是什么？
- **路径依赖分析**：作者是否陷入了巨头的公关叙事？

## 3. 结构化输出协议
- **SCQA 叙事驱动**：
    - **S (Situation)**：目前大家都觉得 [现状]。
    - **C (Conflict)**：但是 [变量] 发生了变化。
    - **Q (Question)**：这是否意味着 [传统逻辑] 已失效？
    - **A (Answer)**：我们的深度洞察是 [算力冗余的核心结论]。
