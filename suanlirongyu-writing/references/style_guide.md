# 「算力冗余」内容风格宪法 v2.0 (冷峻极客风)

## 1. 核心混合逻辑
- **机器之心之骨 (Structure)**：追求极致的严谨性、数据驱动、信源透明（必须引用论文/官方文档）。
- **硅星人之魂 (Narrative)**：追求临场感、第一人称视角（“我们发现”）、以及“说人话”的工业类比。
- **算力冗余之眼 (Insight)**：追求穿透现象，从底层技术逻辑（算力、能效、成本）推演商业终局。

## 2. 句式与表达规则
- **1-3-1 节奏 (Rhythm)**：
    1.  **开场 (Impact)**：一句简短有力的洞察，打破读者的常识预期。
    2.  **拆解 (Deep Dive)**：三句层层递进的硬核分析，包含数据或参数。
    3.  **回响 (Reflection)**：一句冷峻的反思，指出当前趋势的潜在脆弱性。
- **禁用词表 (Negative Constraints)**：
    - 严禁：总之、综上所述、实际上、震惊、史诗级、不得不说、我们看到。
    - 替代：转而使用“逻辑在于”、“变量已经变化”、“如果剥离表象”。

## 3. 类比法则 (Analogy Protocol)
- **准则**：严禁使用自然界（如：种子、河流）或生活类（如：炒菜、穿衣）比喻。
- **允许**：仅允许使用**工程、工业、计算机底层逻辑**进行类比（如：内存溢出、流水线阻塞、热失控、冗余备份、带宽限制）。

## 4. 排版与审美 (Aesthetics)
- **极简专业**：正文使用 Markdown 列表和粗体强调核心逻辑，避免使用花哨的边框或引导语。
- **信息分层**：长难逻辑通过分点（1. 2. 3.）展开，每 500 字必须设置一个小标题（Heading 3）。
