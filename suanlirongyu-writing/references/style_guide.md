# 「算力冗余」内容风格宪法 v4.1 (视觉与叙事升级版)

## 1. 视觉审美规范 (Aesthetics v4.1)
追求“极简工业风”与“智性呼吸感”。
- **H2+Emoji 破局**：每个大章节标题必须使用 `## 🧠 标题` 格式，上方留两行，下方留一行。
- **4行原则**：单个段落严禁超过 4 行文字。超过则必须通过换行或列表拆解。
- **单句锚点**：核心结论、反问、转折必须独立成段，并加粗。这是读者的“阅读抓手”。
- **代码块/术语标记**：首次出现的硬核术语使用 `行内代码块` 格式，保持专业质感。

## 2. 叙事半径规划 (Narrative Scale)
Agent 在大纲阶段必须根据选题性质，自动匹配以下叙事规模：
- **[微观拆解] (800-1200字)**：聚焦单一技术点或热点。节奏极快，结论先行。
- **[中观推演] (2000-3000字)**：分析厂商博弈、产品迭代。包含数据对比、SCQA 结构完整。
- **[宏观叙事] (5000字+)**：探讨底层逻辑变迁或行业终局。要求多维度跨界联想，长短句错落极强。

## 3. 信息密度隔离 (Density Isolation)
- **密集区**：逻辑推演部分，允许短段落连续，但必须用列表（List）进行视觉缓冲。
- **隔离区**：每完成一个逻辑块，必须使用 `---` 分割线，并配合一个引用块（`>`）进行“存盘总结”。

## 4. 视觉引导示例 (The Signature Look)
> **[导语]**：使用引用块展示文章的核心逻辑指纹。

**关键结论独立成行，且前后留空。**
通过加粗形成“第二层阅读线”，即只看加粗部分也能读懂 60% 的内容。
